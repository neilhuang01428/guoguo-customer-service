# 8 種發想流程（依 batch_goal 選用）

## W1｜廣度探索衝刺

- **最適合**：剛開始建 1000 篇地圖、需要大量但不重複的候選
- **步驟**：
  1. 定義產品範圍與受眾，不先下標題
  2. 從六個發想家族各抽 2 種引擎
  3. 每個引擎產 15–20 個問題原子
  4. 按根本答案聚類與去重
  5. 通過可驗證、可行動、值得獨立成文三道門
- **輸出**：約 180–240 個原始候選，收斂為 30–60 個可研究提案。
- **護欄**：不同關鍵字但答案相同，算同一題；禁止只換型號灌水。

## W2｜證據驅動雷達

- **最適合**：OS 更新、新機發布、維修方案與功能可用性變動
- **步驟**：
  1. 蒐集官方變動來源與查核日
  2. 抽出『舊答案 → 新答案』差分
  3. 列出受影響型號、地區、人物與任務
  4. 產生新文、改寫舊文、只加註三種候選
  5. 以第二來源或實機驗證高風險結論
- **輸出**：一份有『為何現在寫』與更新觸發的時效題單。
- **護欄**：沒有差分就不硬寫新聞；海外功能不得默認台灣可用。

## W3｜產品群集建圖

- **最適合**：要把 iPad、iPhone 或 Mac 做成完整主題中心
- **步驟**：
  1. 選一個產品家族
  2. 用歷程 × 搜尋意圖建立熱圖
  3. 標出內容已覆蓋、偏弱與空白
  4. 由空白格運行症狀、比較、限制與迷思引擎
  5. 設計一篇 Hub 與可互鏈的單一問題文章
- **輸出**：一個避免互搶關鍵字、可持續擴張的內容群集。
- **護欄**：Hub 負責導航；單篇只解一個核心問題或決策。

## W4｜售前異議工廠

- **最適合**：福利機選購、價格與信任問題
- **步驟**：
  1. 匯入真實銷售對話
  2. 把表面異議改寫成風險問題
  3. 區分能用規格回答、需檢測證據、需政策承諾
  4. 生成支持購買與不建議購買兩側題目
  5. 評估是否能提供原創照片、測試或保固證據
- **輸出**：既能幫讀者判斷、又不會像廣告的選購內容。
- **護欄**：結論可以是『不適合買』；商業利益不能壓過讀者適配。

## W5｜故障排除工廠

- **最適合**：不能用、變慢、發熱、耗電、斷線等疑難雜症
- **步驟**：
  1. 保留顧客原始症狀
  2. 按範圍、時間、觸發、環境做分叉
  3. 安排低風險且資訊量高的檢查順序
  4. 加入資料保護與停止自行處理點
  5. 再拆成快速急救、深入原理與預防三種內容
- **輸出**：症狀清楚、有安全順序、能真正排除的題目群。
- **護欄**：不先假設根因；不把送修前高風險操作寫成第一步。

## W6｜新舊機決策實驗室

- **最適合**：福利機常見的跨世代、跨級距選購
- **步驟**：
  1. 先定義讀者任務、預算與預計使用年限
  2. 只列會改變任務結果的差異
  3. 建立值得加價與不必加價門檻
  4. 加入 OS／App／維修與配件的長期限制
  5. 分別產出誰適合、誰不適合、何時升級
- **輸出**：從規格表轉成可執行的購買判斷。
- **護欄**：禁止用『新的一定比較好』代替使用情境。

## W7｜退貨與售後回饋環

- **最適合**：已有訂單、客服與退換貨資料的電商
- **步驟**：
  1. 每週抽樣顧客原話與退貨原因
  2. 分期待落差、操作障礙、資訊缺口、商品問題
  3. 尋找可在購買前預防的內容
  4. 發布後觀察是否減少同類詢問
  5. 把新問法與例外回寫成下一輪候選
- **輸出**：直接連到真實營運痛點的高價值題庫。
- **護欄**：移除個資；個案只當線索，達到重複或有證據才泛化。

## W8｜配額式反偏誤輪替

- **最適合**：AI 一直生成同類題目、題庫看起來很多其實很窄
- **步驟**：
  1. 為產品、歷程、意圖與發想家族設最低配額
  2. 每批明確排除上一批過度出現的模式
  3. 要求至少 30% 題目來自限制、接縫、邊界或負面建議
  4. 以語意與根本答案雙重去重
  5. 用熱圖決定下一批，不追求平均灑滿
- **輸出**：更有跨度、比較不像一般 SEO 題庫的候選組合。
- **護欄**：配額用來逼出視角，不代表每個格子都值得寫。
